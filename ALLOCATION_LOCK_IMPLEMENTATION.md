## ✅ 實現方案

### 核心機制：三層防護

#### 層次 1: 自動配發確定性
```python
# convert_with_emptyloc.py - 行 290-294
to_alloc = [p for p in regular_by_qty[skip_n:]]
to_alloc.sort(key=lambda p: (-p['qtyPcs'], -p['qtyBox']))
loc_iter = iter(empty_locs)
for p in to_alloc:
    p['destLoc'] = next(loc_iter, '')
```

✓ **大板優先原則**：按數量由大到小配儲位，小板推後
✓ **無歧義順序**：走道已按空位數排序，同走道按編號遞增
✓ **無重複分配**：每個空儲位只配一次（iter 消耗逐一）

---

#### 層次 2: 配發批次追蹤
```python
# convert_with_emptyloc.py - 行 308-322
for p in pallets:
    if p in to_alloc:
        alloc_counter += 1
        p['alloc_batch_id'] = f"BATCH_{alloc_counter:05d}"
        p['alloc_timestamp'] = alloc_timestamp
```

✓ **唯一批次 ID**：BATCH_00001 ~ BATCH_00282
✓ **時間戳追蹤**：記錄配發時間（精確到秒）
✓ **狀態標記**：BATCH_* = 已配，PENDING_* = 待補，AIRCON = 恆溫

---

#### 層次 3: 邊界鎖定與簽核
```
配發簽核報告 (TXT)
  └─ 全局摘要：FIRST_orderId → LAST_orderId
              FIRST_location → LAST_location

配發鎖定簽核單 (TXT)
  └─ 三級簽核欄位（配發人、驗收人、主管）
  └─ 法律約束：「本配發單據已鎖定，不得追加或修改」

配發對照表 (CSV)
  └─ 282 行完整映射：orderId → destLoc
  └─ 倉管可快速驗證任何貼紙的儲位
```

✓ **法律約束力**：簽核後即為最終狀態
✓ **完整可追蹤**：每筆記錄包含時間、批次、驗證資訊
✓ **快速查詢**：對照表支援秒級查詢

## 🔒 「鎖定」保障機制

### 防重複配發
```
驗證方式：
✓ 配發對照表無重複 destLoc
  grep -o '^[^,]*,[^,]*' 0627_棧板資訊_配發對照表.csv | sort | uniq -d
  (若無輸出 → 無重複 ✓)

✓ 每個空儲位最多配一次
  [原理] loc_iter = iter(empty_locs); next(loc_iter, '')
         遍歷器只能逐一消耗，不會重複
```

### 邊界明確
```
簽核單已確認：
✓ 首張：lotte20260627-1001
✓ 末張：lotte20260627-9040
✓ 首儲位：BE024
✓ 末儲位：BP042

範圍內所有貼紙已配發，範圍外無配發
→ 接續列印清楚知道起點和終點 ✓
```

### 不可更改
```
法律約束：
簽核單明確註明：「本配發單據已鎖定，不得追加或修改」

技術保障：
- 配發對照表是 CSV 靜態檔案，倉管持有
- 若要變更，必須重新執行 convert_with_emptyloc.py 產生新批次
- 舊批次標記為「已完成」，新批次自動遞增編號
→ 確保配發歷史可追蹤 ✓
```

### 完整追蹤
```

✓ orderId        (lotte20260627-1001)
✓ destLoc        (BE024)
✓ alloc_batch_id (BATCH_00001)
```

## 📞 常見問題

### Q: 為什麼有 27 張待補位？
**A**: 常溫倉 309 張 > 空位 282 格，缺口 27 張。
系統自動選擇最小的 27 張暫不配，倉管稍後補配。

### Q: 接續列印從哪裡開始？
**A**: 簽核單明確標示：
- 首張：lotte20260627-1001
- 末張：lotte20260627-9040
  
接續批次會產生新編號 (lotte20260627-1041 起)，邊界清楚。

### Q: 能否修改已配發的儲位？
**A**: **不能**。簽核完成後即「鎖定」，無法變更。
若要修改，必須重新轉換產生新批次。

### Q: 配發對照表如何使用？
**A**: 倉管可用 grep 快速查詢：
```bash
grep "lotte20260627-1234" 0627_棧板資訊_配發對照表.csv
# 結果：lotte20260627-1234,BL234,...
```

### Q: 如何驗證無重複配發？
