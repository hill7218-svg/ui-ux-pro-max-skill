## 改造內容摘要

### 1. UI 架構重組

原架構 (v5):
S0 (開工) → S1 (掃描) → S2 (棧板列表) → S3 (封板) → S4 (彙總)

新架構 (v6.9):
S0 (主選單 A/B/C) 
  ├─ A → S1 (移倉點驗：掃描+新建+加入)
  ├─ B → S2 (核查功能：棧板列表+詳細對點+集成面板)
  ├─ C → S7 (修改數量：PIN驗證+編輯全數據)
  └─ ⚙️ → S6 (開工設定：車號+走道+趟次選擇)

### 2. 各頁面改造細節

**S0 - 新首頁選單**
- 替代原S0的開工設定
- 三個大按鈕：A移倉點驗、B核查功能、C修改數量
- 下方「⚙️ 開工設定」按鈕進S6

**S1 - 移倉點驗（保留核心邏輯，UI簡化）**
- 保留：掃描、數量調整、棧板選擇、新建棧板、加入
- 移除：拍照、QR Code、Google Sheets 同步按鈕
- 簡化UI，讓使用者專心掃描

**S2 - 核查功能（整合原S2+S3+集成功能）**
- 上半部：棧板列表（原S2）
  - 可點選棧板進入詳細對點（S3）
  - 可確認封板
- 下方固定區域：集成功能
  - 📷 拍照上傳（4個按鈕）
  - 🔳 QR Code 顯示
  - ☁️ 同步 Google Sheets

**S3 - 詳細對點（保留邏輯，由S2進入）**
- 封板狀態選擇（正常/異常/破損/混雜/其他）
- 自動分配儲位
- 確認封板

**S4 - 趟次彙總（無改變）**
- 統計和異常檢查
- 關單+上傳

**S6 - 開工設定（原S0邏輯）**
- 車號選擇（含「已完成趟次」提示）
- 主走道選擇（BA-BP + 儲位可用提示）
- 溢出走道（條件式顯示）
- 趟次選擇（第1/2/3趟）
- 尾數趟次勾選

**S7 - 修改數量（全新頁面）**
- PIN 碼驗證（預設9999）
- 通過後顯示所有已封板棧板
- 可編輯品項/數量/狀態

### 3. 新增全域狀態管理

**CLOSED_TRIPS** (司機+趟次記憶)
```javascript
{
  "BBB-2262": [1, 3],     // 已完成第1、3趟
  "BXC-3351": [1, 2, 3]   // 全部完成
}
```
作用：
- 防止重複開工
- 下次選該司機時，自動禁用已完成的趟次
- 顯示「已完成趟次：1,3」提示

**ST.usedCells** (儲位扣減)
```javascript
{
  "BA": 25,  // 已用25格（共72格）
  "BB": 0,
  ...
}
```
作用：
- 替代舊的 locIdx
- 每次封板自動 +1
- S6 顯示「BA 可用 47 / 共 72 格」

**operationLog** (完整操作記錄)
```javascript
[
  {
    timestamp: "2026-06-13 14:30:45",
    action: "掃描" | "新建棧板" | "確認封板" | "拍照" | "開始點驗",
    details: { truck: "BBB-2262", pallet: "PA001", ... },
    syncedToSheet: false
  },
  ...
]
```
作用：
- 記錄每個操作時間戳+詳情
- 關單時一併上傳Google Sheets「操作記錄」頁面

### 4. 新增 Google Sheet 頁面配置

GAS `doPost()` 接收三種 tab 類型：

1. **移倉記錄**
   - 時間戳, 車號, 趟次, 棧板ID, 位置, 品項數, 總件數, 狀態

2. **入庫單**
   - 時間戳, 車號, 棧板ID, 品號, 商品名, 數量, 狀態

3. **操作記錄** (新增)
   - 時間戳, 車號, 操作類型, 詳細資訊, 備註

### 5. localStorage 同步機制

保留的本地資料：
- `ST`: 棧板+品項狀態
- `CLOSED_TRIPS`: 已關單司機+趟次
- `operationLog`: 操作日誌

用途：
- 離線支援：沒網路時可繼續使用
- 離開 APP 後恢復狀態
- 稍後自動上傳同步

### 6. 核心邏輯改進

**確認封板流程** (confirmAndSealPallet)
1. 驗證狀態已選
2. 自動分配儲位：pool[used] → pallet.loc
3. 更新 ST.usedCells[aisle] += 1
4. 標記 CLOSED_TRIPS[truck].push(trip)
5. 記錄 operationLog
6. 清空 ST.curPallet（S1可繼續新建）
7. 刷新 S2 棧板列表

**開工設定流程改進**
- onTruckChange() 檢查 CLOSED_TRIPS[truck]
- 若已完成某趟，該趟按鈕 disabled + 顯示「✓ 已完成」
- pickTrip() 自動跳到「第一個可用趟次」

**PIN 驗證** (S7)
- pinKeyPress() 輸入 0-9
- pinInput.length === 4 時允許 confirm
- 正確返回 PIN_CODE === '9999' → 顯示編輯面板
- 錯誤提示 toast + 清空
