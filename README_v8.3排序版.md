## 📌 三個必用檔案

### 1. 🖨️ 列印系統 (必備)
**檔案**: `ALLY_LOTTE_QR_PRINTER_v8.3_排序版.html`

**功能**:
- ✅ 自動排序 (SKU + 件數)
- ✅ 生成板標貼紙
- ✅ CSV 匯出 (已排序)
- ✅ 可調排版編輯

**快速使用**:
```
1. 用瀏覽器開啟此檔案
2. 左側選擇車次 (如: Air-con6/22-1)
3. 點「① 產生貼紙」
4. 看到排序後的貼紙
5. 按「② 🖨️ 列印/PDF」或「⬇ CSV 匯出」
```

**新增功能亮點**:
```
原本: 依掃描順序列印 (無序)
現在: 自動排序
  - 按 SKU 貨號 A-Z 字母順序
  - 同一 SKU 內按件數從小到大
  - 入庫單號自動重新編號 (20260622-001/002/...)
```

### 2. 📱 拍照 APP (現場測試版)
**檔案**: `ALLY_LOTTE_移倉_v8_測試.html`

**功能**:
- ✅ QR 掃描 (或手動輸入)
- ✅ 自動顯示板資訊 (SKU/品名/件數)
- ✅ 4 格拍照 (相機/檔案上傳)
- ✅ 進度追蹤
- ✅ 模擬上傳 (測試模式)

**快速使用**:
```
1. 用瀏覽器開啟此檔案
2. 輸入車號 (如: BBB-2262) → 開始拍照
3. 掃 QR Code 或輸入 orderId (如: 001)
4. 系統自動顯示板資訊
5. 點 4 個照片槽拍照
6. 點「✅ 完成此棧板」
7. 重複 3-6 多次
8. 點「⬆ 上傳所有」→ 測試上傳
```

**測試模式說明**:
- 🟢 不需要 Google Drive 帳號
- 🟢 可直接在現場測試
- 🟢 上傳為模擬 (控制台日誌記錄)
- 🟢 完整的元資料綁定 (orderId + timestamp)

## 📞 遇到問題？

### 列印系統
- **貼紙順序不對**: 檢查「排序後的 orderId 是否遞增」
- **字體太小**: 在「排版控制」調整倍數滑桿
- **無法列印**: Ctrl+P 開啟瀏覽器列印對話

### APP
- **掃描不工作**: 確保瀏覽器相機權限已開啟
- **拍照失敗**: 嘗試「上傳圖片」代替
- **上傳卡住**: 等待 30 秒 (模擬 API 延遲)

### CSV
- **無法下載**: 關閉廣告攔截器
- **格式問題**: 用 Excel 或 Numbers 開啟檢查

**詳細故障排查** → 見 `使用說明_v8.3排序版.md`

## 💡 使用建議

1. **列印前預覽**
   - 先產生 3-5 張貼紙預覽排序
   - 確認無誤後再列印全量

2. **CSV 匯出時機**
   - 產生貼紙後立即匯出
   - 或用於系統批量導入

3. **APP 測試流程**
   - 先單張板測試 (1-2 張)
   - 確認拍照/上傳順暢後再批量
   - 在模擬模式下測試完全流程

4. **版本對應**
   - 貼紙上的 orderId 必須與 CSV 中的一致
   - APP 中掃描的 orderId 也必須相同
