## 0. 一句話總覽

```
客戶訂單 XLSX ──(convert_customer_xlsx.py)──> 標準 CSV/XLSX
        └─> 匯入 v8.3 列印器 ─> 列印 10×17cm 貼紙 + 匯出 WMS 入庫單
                  └─> 貼紙 QR ─> photo.html 掃描 ─> 拍照上傳 Google Drive
```

三個核心檔案（都在 `ALLY_LOTTE_WH/APP/` 與專案根目錄）：
| 檔案 | 角色 |
|------|------|
| `convert_customer_xlsx.py` | 把客戶訂單轉成列印器可吃的格式 |
| `ALLY_LOTTE_WH/APP/ALLY_LOTTE_QR_PRINTER_v8.3_排序版.html` | 列印貼紙 + 匯出入庫單 |
| `ALLY_LOTTE_WH/APP/photo.html` | 掃 QR、拍照、上傳 Google Drive |

## 1. 快速操作流程（明天直接照做）

### Step 1：轉換客戶訂單
```bash
python3 convert_customer_xlsx.py <客戶訂單.xlsx> <商品主檔.xlsm> [輸出路徑]
# 不帶參數則用程式內 DEFAULT_* 預設路徑
# 輸出：customer_converted.csv 與 customer_converted.xlsx
```
看終端機輸出的「各車次板數 / 贈品板數 / 無條碼板」做 sanity check。

### Step 2：列印貼紙
1. 瀏覽器開 `ALLY_LOTTE_QR_PRINTER_v8.3_排序版.html`
2. 「匯入資料」→ 選 `customer_converted.xlsx`（或 .csv）
3. 確認顯示「✅ 已匯入 N 筆」，下拉變成「全部 N 板」
4. 選車次、設張數 → 「產生貼紙」→「列印/PDF」

### Step 3：匯出入庫單（WMS）
- 點「⬇ 匯出入庫單 XLSX（WMS可匯入）」→ 最後一欄已含「儲位」

### Step 4：現場拍照驗收
- 手機開 GitHub Pages 的 `photo.html`
- 掃貼紙 QR → 自動帶出入庫單號 + 商品數量 → 變焦對焦 → 拍照 → 上傳 Drive
- 檔名格式：`入庫單號_序號_拍攝時間.jpg`

## 2. 客戶訂單格式（輸入）

客戶 XLSX 有兩個分頁：**總表**（不參考）、**工作表1**（明細，實際讀這個）。

`工作表1` 欄位（程式用固定 index 抓）：
| index | 欄位 | 用途 |
|-------|------|------|
| 0 | 儲位 | **唯一棧板識別**（去重 key），輸出為 sourceId |
| 1 | 品號 | sku，比對商品主檔 |
| 2 | 品名 | name |
| 3 | 數量 | 銷售單位數量 → 加總成 qtyPcs（件） |
| 4 | 單位 | 箱/盒/個/包/瓶/雙…（混合） |
| 6 | 大 | **客戶已換算好的箱數**（單位恆為「箱」）→ 加總成 qtyBox |
| 12 | 到期日 | expDate |
| 16 | 移倉 | trip（如 `6/23-4`、`6/23空調-1`） |

### ⚠️ 關鍵雷區：板數 ≠ 筆數
- 客戶總表是用「筆數」計，一個儲位常有多筆明細（同品項不同效期/批號）。
- **正解：每個唯一「儲位」= 一板**。722 筆 → 去重後 207 板。
- XYZ 報表類品號筆數特別多，更要靠儲位去重。

### ⚠️ 箱數換算
- 不用自己算！客戶 `大` 欄已分解好（大=箱、中/小=零頭），且單位恆為「箱」。
- **qtyBox = 該儲位所有列的「大」欄加總**。
- `大=0` 的是不足一箱的散件贈品（以「個」計，箱數 0）。

## 3. 商品主檔（條碼來源）

檔案：`a0d64486-...jtu...xlsm`，多分頁。程式用兩層查找：
1. **`DSV系統 商品資料`**（最全，5000+ 筆）：`條碼`欄=單品條碼、`箱條碼`欄=外箱條碼。
2. **各國/贈品分頁**（jtu summary、泰/韓/日/越_商品、禮盒、贈品+陳列架…）：`產品條碼`、`外箱條碼`、`產品細分類`（判斷贈品）。

### 條碼選用規則（國際條碼）
| 商品類型 | 用哪個條碼 | 判定方式 |
|----------|-----------|---------|
| **贈品** | 單品條碼（最小單位） | `產品細分類=贈品` 或 SKU 開頭 `SW` 或品名含「贈品」 |
| **一般商品** | 外箱條碼 | 其餘 |

- 條碼雜訊值（`NO`/`N/A`/`0`/`#VALUE!`/空白/中文）一律視為「無條碼」→ 貼紙顯示「—」。
- 無條碼板：XYZ 報表/信封/手提袋、WW 墊箱/陳列盒、部分無條碼贈品變體（SW…-1/-2）。這些**主檔本來就沒條碼**，正常現象。

## 4. 標準輸出格式（轉換器 → 列印器）

CSV/XLSX 共 **10 欄**，列印器匯入驗證會檢查這些欄位名：
```
sourceId, sku, name, qtyBox, qtyPcs, srcLoc, destLoc, trip, expDate, ean
```
| 欄位 | 內容 | 備註 |
|------|------|------|
| sourceId | 儲位 | 唯一識別、QR 的 loc 參數 |
| sku | 品號 | |
| name | 品名 | |
| qtyBox | 箱數 | =大欄加總 |
| qtyPcs | 件數 | =數量加總 |
| srcLoc | （空） | 客戶要求不帶來源儲位 |
| destLoc | （空） | 上架儲位，列印器內配發 |
| trip | 移倉車次 | |
| expDate | 到期日 | |
| ean | 國際條碼 | 贈品=單品、其他=外箱 |

## 5. v8.3 列印器重點

### 入庫單號（orderId）格式
```
lotte + 2026MMDD + 趟次代碼 + 流水號(每車次從001起)
例：lotte202606230623-4001        (6/23-4 第1板)
   lotte202606230623AIRCON-1001  (6/23空調-1 第1板)
```
- 日期、趟次代碼都從 `trip` 自動解析（`6/23-4` → 日期 0623、代碼 `0623-4`）。
- **空調 / 恆溫 一律轉成 `AIRCON`**（見雷區）。
- 限制 **30 字內**，超過自動截斷；並過濾非 ASCII 字元。
- 程式位置：`parseTrip()` / `buildOrderId()`（在 sort 函式上方）。

### QR Code 內容
```
{PHOTO_PAGE_URL}?id={入庫單號}&loc={儲位}
```
- `id`（入庫單號）→ photo.html 拿來當**檔名**。
- `loc`（儲位）→ photo.html 拿來**查商品數量**（比對棧板資訊表的 sourceId 欄）。

### 貼紙尺寸
- **10cm × 17cm**（100mm × 170mm，直式）。CSS `.sticker` width/height。

### 各欄位用途
| 貼紙元素 | 內容 |
|----------|------|
| 入庫單號文字 + 條碼 | orderId（新格式） |
| 國際條碼（文字） | ean |
| 來源儲位 | srcLoc（目前空） |
| QR Code | id=orderId&loc=sourceId |
| 來源單號（底部） | sourceId（儲位） |
| 上架儲位 | destLoc（列印器內配發） |

### WMS 入庫單匯出
- 22 欄官方範本 + **第 23 欄「儲位」**（=sourceId）。
- `palletToWmsRow()` / `WMS_HEADERS`。

## 6. photo.html 重點

- **掃描模式**：jsQR 即時掃貼紙 QR；也相容 `?id=` URL 直開。
- 掃到後解析 `id`（入庫單號）與 `loc`（儲位）：
  - `APP.orderId` = 入庫單號 → 檔名、Sheet 記錄、畫面顯示
  - `APP.sourceId` = 儲位 → `fillPalletInfo()` 查商品數量
  - 舊版 QR（只有純文字或 `?id=儲位`）：無 `loc` 時 orderId/sourceId 同值，仍可運作。
- **變焦/對焦**：硬體變焦（Android, `track.applyConstraints`）+ 軟體變焦 fallback（iOS, CSS scale + canvas 裁切）；點畫面對焦光圈。
- **拍照**：3 秒倒數 → 截圖 → 上傳 Google Drive。
- **檔名**：`{入庫單號}_{序號}_{拍攝時間}.jpg`（沿用 6/22 邏輯，只把識別碼換成入庫單號）。
- 後端：Google Apps Script（`CONFIG.webAppUrl`），`uploadPhoto` 存 Drive、`append` 寫 Sheet。

## 7. 雷區清單（踩過的坑，務必記得）

1. **CODE128 條碼不能編碼中文** → 入庫單號裡的「空調」會讓條碼產生失敗（空白）。已用 `空調/恆溫→AIRCON` + 過濾非 ASCII 解決。**明天若有其他中文車次名，要再加對應英文。**
2. **XLSX `sheet_to_json` 會略過「整欄全空」的欄位** → srcLoc/ean 全空會導致匯入驗證說「缺欄位」。已用 `{defval:''}` 解決。（CSV 不受影響。）
3. **`PALLET_DATA` 原本是 `const`** → 匯入時重新賦值會報「Assignment to constant variable」。已改 `let`。
4. **板數要用儲位去重**，不是筆數。
5. **箱數用「大」欄**，不要自己用件數除箱入數（客戶已算好）。
6. **orderId 是列印時動態產生**，不存在於 CSV/匯出檔；photo.html 靠 QR 直接帶，不靠查表。
7. **分支注意**：變焦版 photo.html 在 `main`；若從舊基底開分支會缺變焦，需 `git checkout main -- ...photo.html` 取回。

## 8. 明天可重複性確認（DOUBLE CHECK）

✅ **可以重複使用**，條件：
- 客戶訂單沿用同結構（總表 + 工作表1，且工作表1 欄位順序相同）。
- 商品主檔沿用同檔（或更新版，分頁名/欄位名不變）。
- 轉換器已支援命令列參數，換檔案路徑即可。
- orderId 日期/車次自動從 `trip` 解析，**換日期免改程式**（6/24 → 自動 20260624）。
- 列印器匯入格式固定，轉換器輸出固定 → 對得上。

⚠️ **需要人工確認的情況**：
- 出現**新的中文車次關鍵字**（非「空調/恆溫」）→ 要加英文對應到 `parseTrip()`。
- 商品主檔**分頁名稱或欄位標題改變** → 要調整 `build_master_lookup()`。
- 客戶**工作表1 欄位順序改變** → 要調整 `main()` 裡的固定 index（I_LOC/I_SKU…）。
