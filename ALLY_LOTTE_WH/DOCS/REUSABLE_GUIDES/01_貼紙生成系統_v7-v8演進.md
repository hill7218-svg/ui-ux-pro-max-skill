## 🔄 版本演進路線圖

### v7.0 — 基礎雙條碼系統（基線）

**核心概念**：來源單號 + 儲位配置

```
來源單號 QR Code (碼A)        儲位 Code128 條碼 (碼B)
格式: ALLY-YYYYMMDD-[AB]-[1-3]-[001-192]   格式: BA012 / BA013 / ... / BP072
```

**關鍵代碼結構**：

```javascript
// 資料結構 (v7.0)
const PALLET = {
  id: "ALLY-20260622-A-1-001",           // 來源單號
  sku: "BK9275",                          // 商品 SKU
  qtyBox: 80,                             // 箱數
  qtyPcs: 3200,                           // 件數
  sourceLocation: "C05A0194-0",           // 來源儲位
  destLocation: "BA012",                  // 上架儲位（待綁定）
  scanned: false
};

// HTML 貼紙生成 (v7.0)
function drawPalletLabel(pallet) {
  // 1. 來源單號 (文字 + Code128條碼)
  // 2. 商品基本資訊 (SKU/品名/箱數)
  // 3. 上架儲位 (文字 + Code128條碼)
  // 4. 簡單 QR Code (純文本：sourceId)
}
```

**優勢**：簡單、離線運作、無 API 依賴  
**限制**：資訊量少、無排序邏輯、上架儲位需手動預配

```javascript
function generateLabels_v8_2(palletList, startIndex = 1, count = 5) {
  const filteredList = palletList.slice(startIndex - 1, startIndex - 1 + count);
  
  filteredList.forEach((pallet, idx) => {

      width: 2,
      height: 50

      text: JSON.stringify({
        s: pallet.sourceId,
        o: pallet.orderId,
        k: pallet.sku,
        d: pallet.destLocation
      }),
      width: 128,
      height: 128,
      colorDark: "#000000",
      colorLight: "#ffffff",
      correctLevel: QRCode.CorrectLevel.H
    });
  });
}
```

### v8.3 — 生產版（排序 + CSV 匯出 + 拍照驗收）

**新增功能**：
- 🔀 **自動排序** (SKU 字母序 + 件數遞增)
- 📥 **CSV 匯出** (用於 WMS/OMS 導入)
- 📷 **拍照驗收** (集成 photo.html)
- 🔄 **手動排序** (允許拖拽重排)
- 💾 **CSV 匯入** (支援修改後的資料重新匯入)

**核心實現**：

```javascript
// v8.3 排序演算法
function sortPalletsBySKUandQty(pallets) {
  let sorted = JSON.parse(JSON.stringify(pallets));
  
  // 按 SKU 字母順序 + 件數遞增排序
  sorted.sort((a, b) => {
    let skuCmp = a.sku.localeCompare(b.sku);    // A-Z 排序
    if (skuCmp !== 0) return skuCmp;
    return a.qtyPcs - b.qtyPcs;                 // 同 SKU 按件數小→大
  });
  
  // 重新分配 orderId (入庫單號)
  sorted.forEach((p, idx) => {
    p.orderId = '20260622-' + String(idx + 1).padStart(3, '0');
  });
  
  return sorted;
}

// v8.3 CSV 匯出
function exportToCSV(pallets) {
  const headers = [
    'sourceId', 'sku', '品名', '箱數', '件數', 
    '上架儍位', '來源儍位', '效期', '趟次', '棧板號', '入庫單號'
  ];
  
  const rows = pallets.map(p => [
    p.sourceId,
    p.sku,
    p.name,
    p.qtyBox,
    p.qtyPcs,
    p.destLocation,
    p.sourceLocation,
    p.expiry || '',
    p.trip || '',
    p.no,
    p.orderId
  ]);
  
  // 轉換為 CSV（處理引號等特殊字元）
  let csv = headers.map(h => `"${h}"`).join(',') + '\n';
  csv += rows.map(row => 
    row.map(cell => `"${String(cell).replace(/"/g, '""')}"`).join(',')
  ).join('\n');
  
  // 下載
  const blob = new Blob([csv], { type: 'text/csv;charset=utf-8;' });
  const link = document.createElement('a');
  link.href = URL.createObjectURL(blob);
  link.download = `OMS_入庫單_${Date.now()}.csv`;
  link.click();
}

// v8.3 CSV 匯入
function importFromCSV(file) {
  const reader = new FileReader();
  reader.onload = (e) => {
    const csv = e.target.result;
    const lines = csv.split('\n');
    const headers = lines[0].split(',').map(h => h.replace(/"/g, ''));
    
    const pallets = [];
    for (let i = 1; i < lines.length; i++) {
      if (!lines[i].trim()) continue;
      
      const values = parseCSVLine(lines[i]); // 自訂函數處理引號
      const pallet = {};
      headers.forEach((h, idx) => {
        pallet[h] = values[idx];
      });
      pallets.push(pallet);
    }
    
    return pallets;
  };
  reader.readAsText(file);
}
```


## 📋 常見問題

### Q1: 何時該用 v7.0？
A: 資訊簡單、預算低、不需詳細商品資訊。限 v7.0 開發時期使用。

### Q2: 何時升級到 v8.2？
A: 需要在貼紙上顯示品名、箱數，需要移動端掃碼自動帶出資訊。

### Q3: 何時升級到 v8.3？
A: 需要自動排序、CSV 導入 WMS/OMS、需要拍照驗收。

### Q4: 排序後順序會亂嗎？
A: 不會。v8.3 在排序後重新分配 orderId，並與貼紙順序保持完全一致。

### Q5: CSV 匯出後能再匯入嗎？
A: 可以。修改 CSV 後重新匯入，系統會更新資料。

### Q6: QR Code 如何掃描？
A: 任何手機或 PDA 掃碼 APP 都可。v8.2+ 的 QR Code 內容為 JSON，需後端解析。
