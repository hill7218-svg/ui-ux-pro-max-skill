## 📱 APP 邏輯流程

### 掃描→驗證→封板 完整流程

```
START (S0 輸入車號)
  ↓
S1: 掃條碼
  ↓ (條碼映射查表)
查 BARCODE_MAP[條碼]
  ├─ 找到 → ITEM_MASTER[sku]
  │   顯示: 品名、EAN-13、標準箱數
  │
  └─ 沒找到 → ❌ 錯誤提示
  
  ↓ (輸入數量)
用戶輸入箱數或盒數
  
  ↓ (掃源ID)
掃 PALLETS_6_22 中的 sourceId
  
  ↓ (DOUBLE CHECK)
比較: 掃描商品 SKU vs 源ID對應 SKU
  ├─ 相同 → 🟢 綠色通過
  │   進行封板
  │
  └─ 不同 → 🔴 紅色警告
      不允許封板
  
  ↓ (封板)
1. 查表: LOCATION_MAP[sourceId] → 儲位
2. 顯示 S3 預印儲位
3. 記錄 LOG
4. 扣除 SKU_REMAIN[sku]
5. 計數 +1

✅ 完成 (S4 摘要)
  ├─ 本次掃描統計
  ├─ 操作記錄預覽
  └─ 關閉班次 → 上傳 GAS
```
