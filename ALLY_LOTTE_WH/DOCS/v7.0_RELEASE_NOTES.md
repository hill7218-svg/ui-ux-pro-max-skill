## 🎯 核心流程（使用者確認版）

### 移出端（鶯歌 — 操作員掃貼紙儲位）

```
S0 開工 → 選車號/走道/趟次
   ↓
S1 掃商品 → 掃來源單號QR → 掃貼紙儲位Code128（建議儲位）
   ↓
S3 封板 → 確認儲位、拍異常照、確認封板
   ↓
S4 關單 → 產生入庫單（含最終儲位）→ 上傳 Google Sheet
   ↓
貨車運到瑞芳
```

**操作要點**：
- **貼紙儲位為來源**：掃Code128後 suggestedLoc 被綁定
- **現場可覆寫**：若被佔可手動改儲位（走道面板選位）
- **封板時確定**：最終儲位寫進入庫單

### 收貨端（瑞芳 — S8 軟驗證）

```
S8 卸貨驗收 → 掃來源單號QR（查棧板品項）
   ↓
掃儲位Code128（讀入庫單最終儲位）
   ↓
軟驗證 ✅ 綠燈（一致）
       ⚠️ 黃燈（不符 → 輸入原因）
       ❌ 紅燈（無效）
   ↓
確認上架 → 記錄收貨狀態 → 上傳 Sheet
```

**關鍵**：入庫單最終儲位是**跨裝置權威來源**。

## ✅ v7.0 新增 & 修復

### 新增功能
1. **雙條碼貼紙**
   - 碼A：來源單號 QR（掃一次綁板）
   - 碼B：儲位 Code128（掃一次綁儲位）
   - 不足時印「—」可手寫備援

2. **S8 卸貨驗收**（v7.0 P2）
   - 掃貼紙 QR → 查棧板品項
   - 掃儲位 Code128 → 軟驗證（綠/黃/紅燈）
   - 黃燈時 prompt 捕捉覆寫原因
   - 記錄收貨狀態回 Sheet

3. **S0 主菜單 D 按鈕**
   - 卸貨驗收直達入口（供瑞芳操作員）

4. **工作流改名**（v7.0 P4）
   - UI：核查模式 ↔ 快速模式（去耦車型 A/B）
   - 內部：'inspect' ↔ 'fast'

### 重大修復（v7.0 P5 發版驗收）
1. **入庫單補儲位欄**（根治跨裝置斷鏈）
   - `buildWmsRows` finalLoc 從 dead code → 寫第 23 欄「儲位」
   - S8 expectedLoc 改從入庫單讀取（主基準）

2. **S8 軟驗證邏輯修正**
   - expectedLoc 優先讀 `s8_items[].finalLoc`（跨裝置最終儲位）
   - 本機 `ST.pallets` 僅作同台自驗 fallback
   - 黃燈覆寫時強制 prompt 捕捉原因

3. **pickSt 槽位洩漏修復**
   - 自動分配位快取於 `p._autoLoc`
   - 狀態切換不再重複 assignLoc

4. **assignLoc 哨兵確認**
   - 回傳 '無空格'/'QA-HOLD-FULL' 被正確攔截

## 🔐 前置部署

### 必要配置（Google Sheets + Apps Script）

**1. Google Sheet ID**
```
填入 ALLY_LOTTE_QR_PRINTER_v7.0.html 
- L577：const GOOGLE_SHEETS_ID = "YOUR_SHEET_ID";
```

**2. Sheet 分頁結構**
| 分頁名 | 由誰寫 | 用途 |
|-------|-------|------|
| 儲位清單 | 你維護 | 列印器讀取預配儲位 BA~BP |
| 入庫單 | V6.9 寫 | 23 欄 WMS 格式（含來源單號+最終儲位） |
| 收貨狀態 | V6.9 寫 | S8 驗收記錄（9 欄） |
| 操作記錄 | V6.9 寫 | 時間軸日誌（可選） |
| 司機清單 | 你維護 | 司機姓名/車號對應 |
| 商品清單 | 你維護 | 商品貨號/EAN |

**3. 「入庫單」分頁需新增「儲位」表頭欄** ⚠️ 重要
```
新增第 23 欄表頭 = 「儲位」
(Google Sheets 既有分頁需手動補，GAS 新建分頁會自動帶)
```

**4. Apps Script GAS 後端**（負責 Sheet 讀寫、去重）
- 提供 `Backend.readTab(tabName)` → 讀分頁
- 提供 `Backend.appendRows(tabName, rows)` → 追加列
- 防重複上架去重邏輯
