## P2 — 工具② S8 上架刷碼（移倉對點 APP）

檔案：ALLY_LOTTE_WAREHOUSE_v6.9.html（標題已升 v7.0）

S1 掃雙碼：
- scanPalletLabel()：掃來源單號 QR → 驗格式 → 新建/選取棧板
- scanPalletLocation()：掃儲位 Code128 → 存入 pallet.suggestedLoc
- 新建/選取棧板後自動顯示儲位輸入框等待掃碼

S8 上架流程：
- s8ScanLocation()：刷儲位條碼 → 軟驗證（綠/黃/紅燈）
  · 綠燈：實刷 = 建議位，可上架
  · 黃燈：實刷 ≠ 建議位，需二次確認（記錄覆寫）
  · 紅燈：無效儲位，要求改刷
- s8ConfirmReceiving()：要求先刷儲位通過、記錄 isOverwrite、operationLog

## P3 — assignLoc 預配優先（做法甲）

檔案：ALLY_LOTTE_WAREHOUSE_v6.9.html

修正隱性 bug：先前貼紙預配儲位從未真正生效，封板永遠落到 assignLoc 重算。

改動：
- confirmAndSealPallet 儲位分配鏈改為：
    pallet.loc || pallet.suggestedLoc || ST._pendingLoc || assignLoc()
  優先序：已覆寫 > 貼紙建議 > S3待定 > 自動分配
- assignLoc() 保留為 fallback，加註解「僅在無預配儲位時觸發」
- 破損 QA-HOLD 隔離邏輯仍由 assignLoc 接手，不受影響

實質效果 = 架構圖承諾的「①預配 + V6.9 可覆寫」。

## P4 — 工作流改名（A/B → 核查/快速）

檔案：ALLY_LOTTE_WAREHOUSE_v6.9.html

根因：來源單號的車型 A/B 與 UI 工作流 A/B 撞名，現場與程式都易誤判。

UI 文案：
- 按鈕 #wf-a：「A · 核查後封板」→「🔍 核查模式」
- 按鈕 #wf-b：「B · 直接封板」→「⚡ 快速模式」
- #wf-desc 描述文案改中文模式名

內部值（徹底斷開字母耦合）：
- ST.workflow 預設 'A' → 'inspect'
- setWorkflow('A'/'B') → setWorkflow('inspect'/'fast')
- updateBSealBar：!== 'B' → !== 'fast'
- s3Back：=== 'B' → === 'fast'
- addLog('切換工作流', w) → addLog('切換模式', 核查模式/快速模式)

紅線（未動）：
- 車型 A/B：genSourceId vehicleType（趟1-3=A、4-6=B）
- scanPalletLabel regex /^ALLY-\d{8}-[AB]-[1-3]-\d{3}$/
- buildWmsRows 來源單號第7欄
→ 來源單號契約 ALLY-YYYYMMDD-A-1-001 完全不變
