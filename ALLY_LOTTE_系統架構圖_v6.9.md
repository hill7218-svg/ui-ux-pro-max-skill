## 二、作業時間軸（移出 → 運送 → 收貨）

```
【事前 · office】 🏷️ 工具①
  讀 Sheet 儲位清單(BA~BP 第2~4層)
   → 每板配「建議儲位」
   → 列印 192 張貼紙（QR + 建議儲位）

        ↓ 紙本貼紙帶到鶯歌

【移出 · 鶯歌】 📱 V6.9
  S6 開工(A車趟1 / BN走道)
   → S1 掃商品 → 選倉別(A47…)
   → 封板：顯示貼紙建議儲位
            └ 實際被占 → 覆寫成真實儲位
   → 貼「ALLY-…-A1-001」貼紙到膠膜
   → 累積 32 板 → 關單發車
   → ☁️ 上傳「依板明細」入庫單（含最終儲位）

        ↓ 貨車運到瑞芳

【收貨 · 瑞芳】 📱 V6.9
  S8 卸貨 → 刷貼紙 QR(來源單號)
   → 查 Sheet 該板入庫單 → 驗收 ✓
   → 上架：刷儲位條碼 → 帳務入儲 → 堆高機上架
```

## 三、儲位配發決策（已定案）

**模式：工具①預配 + V6.9 可改**

| 階段 | 誰 | 做什麼 |
|------|----|--------|
| 列印貼紙 | 🏷️ 工具① | 讀儲位清單 → 配「建議儲位」→ 印在貼紙 |
| 現場封板 | 📱 V6.9 | 顯示建議儲位；實際被占可覆寫 → 寫入最終儲位 |

> 好處：貼紙先有儲位（免手寫），現場仍保留彈性覆寫。

## 五、入庫單格式：依板明細（非加總）

每塊棧板 = 獨立資料列，22 欄。同一板多 SKU → 多列共用同一「來源單號」。

- 刷單一板 QR → `WHERE 來源單號 = ALLY-…-A1-001` → 查回該板全部品項
- 收貨驗收用此查詢

> 與舊版「依貨號跨32板加總」不同；那版失去板身份，無法刷單板查詢。
