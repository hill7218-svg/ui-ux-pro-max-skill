## 🎯 核心功能介紹

### 列印系統（v8.3 排序版）

#### 新增：自動排序功能

當您按下「產生貼紙」按鈕時，系統會自動執行以下步驟：

1. **篩選**：取得所選車次（如 Air-con6/22-1）的所有板
2. **排序**：按照兩層邏輯排序
   - **第一層**：SKU 貨號 A-Z 字母順序（遞增）
   - **第二層**：同一 SKU 內，按每板件數從小到大（遞增）
3. **重新編號**：生成新的入庫單號
   - 格式：20260622-001, 20260622-002, ...
   - 基於排序後的位置
4. **生成貼紙**：按新順序印出板標
   - 顯示排序後的入庫單號
   - 保留原 sourceId 用於 QR Code

#### 操作步驟

```
1. 開啟 ALLY_LOTTE_QR_PRINTER_v8.3_排序版.html
2. 左側控制面板：
   - 選擇「行次」(車次)
     例：Air-con6/22-1（共18板）
        或 Regular6/22-1（共35板）
   - 設定「起始號」= 1（通常）
   - 設定「張數」= 要產生多少張貼紙
      預設 3 張，可改成 5/10/18/35 等
3. 按「① 產生貼紙」按鈕
   ⏱️ 系統自動排序並生成
4. 檢視排序結果
   - 第一張貼紙應該是 SKU A-Z 字母序中最小的
   - 同一 SKU 內，件數（右下角 XX件）是遞增的
5. 調整版面（可選）
   - 啟用「拖曳模式」
   - 調整字體倍數、位置等
6. 列印或匯出 PDF
   - 按「② 🖨️ 列印/PDF」
   - 或按「⬇ CSV 匯出（已排序）」導出 CSV
```

#### 排序規則詳解

**範例數據**：假設 Air-con6/22-1 有以下板

| 原序 | SKU | 件數 | 排序後入庫單號 |
|------|-----|------|--------------|
| #001 | BK9275 | 3200 | 20260622-001 |
| #002 | BK9275 | 3200 | 20260622-002 |
| #006 | BK9276 | 960  | 20260622-003 |
| #007 | BK9276 | 480  | 20260622-004 |
| #008 | DK9051 | 320  | 20260622-005 |

**排序邏輯**：
1. ✅ BK9275 (小於 BK9276 字母序)
2. ✅ BK9275 (小於 DK9051)
3. ✅ BK9276 件數 480 小於 960 ← **在同一 SKU 內，件數小到大**
4. ✅ DK9051 (字母序最後)

## 📤 CSV 匯出

### 功能
將排序後的板資訊匯出為 CSV 檔案。

### 操作

```
1. 在「產生貼紙」後

```

## 🔧 測試模式詳解

### 當前狀態（測試版）

APP 目前運作在**測試模式**：
- ✅ QR 掃描功能正常
- ✅ 拍照功能正常（相機 / 檔案上傳）
- ✅ 進度追蹤正常
- ⚠️ Google Drive 上傳為「模擬」（不實際上傳）

### 模擬上傳流程

當您點「⬆ 上傳所有」時：
1. 系統顯示進度條
2. 每個棧板延遲 300-500ms（模擬 API 延遲）
3. 在瀏覽器控制台輸出模擬上傳日誌
4. 顯示「✅ 測試上傳完成」

**控制台輸出範例**：
```
[測試上傳] 20260622-001 - 4 張照片 - 時間戳：2026-06-22T14:30:45Z
[測試上傳] 20260622-002 - 3 張照片 - 時間戳：2026-06-22T14:31:12Z
```

### 切換至生產模式

如需真實上傳至 Google Drive，須：

1. **取得 Google Drive API 密鑰**
   - 至 Google Cloud Console
   - 建立新專案
   - 啟用 Google Drive API
   - 建立服務帳號
   - 下載 JSON 金鑰

2. **修改 APP 程式碼**
   搜尋 `uploadAll()` 函數，將測試的 `console.log` 替換為真實 API 呼叫：
   ```javascript
   // 測試模式（當前）
   console.log(`[測試上傳] ${orderId}...`);
   
   // 生產模式（修改後）
   const response = await fetch('https://www.googleapis.com/upload/drive/v3/files?uploadType=media', {
     method: 'POST',
     headers: { 'Authorization': `Bearer ${accessToken}` },
     body: photoData
   });
   ```

## 📋 技術細節

### 排序演算法

```javascript
function sortPalletsBySKUandQty(pallets) {
  let sorted = JSON.parse(JSON.stringify(pallets));
  sorted.sort((a, b) => {
    // 第一層：SKU 字母序
    let skuCmp = a.sku.localeCompare(b.sku);
    if(skuCmp !== 0) return skuCmp;
    // 第二層：件數小到大
    return a.qtyPcs - b.qtyPcs;
  });
  // 重新分配入庫單號
  sorted.forEach((p, idx) => {
    p.orderId = '20260622-' + String(idx + 1).padStart(3, '0');
  });
  return sorted;
}
```

### 資料不變性

- 原始 `PALLET_DATA` 保持不變
- 排序是暫時的（僅在該次產生時生效）
- 每次「產生貼紙」都會重新排序並重新分配 orderId

## 📞 支援與問題排查

### 列印系統常見問題

**Q: 貼紙順序和預期不同？**  
A: 刷新頁面（F5）後重新產生，或檢查控制台（F12）是否有錯誤。

**Q: CSV 無法匯出？**  
A: 檢查瀏覽器是否允許下載；關閉廣告攔截器。

**Q: 字體太小或太大？**  
A: 在「自由排版」區域調整字體倍數滑桿，然後存檔。

### APP 常見問題

**Q: 掃描不工作？**  
A: 確認瀏覽器已打開相機權限；用 Enter 鍵確認輸入。

**Q: 拍照失敗？**  
A: 檢查瀏覽器權限（設定 → 隱私）；試試用「上傳圖片」代替。

**Q: 上傳進度條卡住？**  
A: 這是測試模式的正常行為，約 30 秒後完成。
