## 一、核心契約（唯一接點）

### 來源單號格式
- **格式**：`ALLY-{YYYYMMDD}-{A|B}-{trip#}-{board#}`
- **例子**：
  - `ALLY-20260614-A-1-001` (6月14日，A車趟1，第1板)
  - `ALLY-20260614-A-2-032` (6月14日，A車趟2，第32板)
  - `ALLY-20260614-B-1-001` (6月14日，B車趟1，第1板)

### 車趟對應規則
| 趟次 | 車型 | tripInType |
|------|------|-----------|
| 1 | A車 | A-1 |
| 2 | A車 | A-2 |
| 3 | A車 | A-3 |
| 4 | B車 | B-1 |
| 5 | B車 | B-2 |
| 6 | B車 | B-3 |

### QR Code 內容
- **文字內容**：Just the 來源單號 (plain string)
  - **不是** JSON
  - **不是** 複雜結構
  - 直接掃描 → 得到 `ALLY-20260614-A-1-001`

### 入庫單格式（per-board）
- **結構**：每板 = 獨立行，多SKU = 多行共同 來源單號
- **關鍵欄位**：
  - 第 7 欄（來源單號）= `ALLY-YYYYMMDD-A-1-001`
  - 第 19 欄（虛擬倉）= 該品項的倉別 (A47, A49, etc.)
- **查詢邏輯**：`SELECT * WHERE 來源單號='ALLY-20260614-A-1-001'` → 回傳該板全部品項

## 四、交叉驗證場景

### 場景 A：列印貼紙 → V6.9 掃描
```
1. Tool① 產生 192 張貼紙（包含QR+來源單號）
   → 列印出來
   
2. V6.9「開工」相同日期/走道/趟次
   
3. V6.9 S1「顯示 QR Code 面板」
   → 掃描列印的貼紙 QR
   
4. 驗證：
   ✅ QR 掃描值 = V6.9 自動產生的棧板 ID
   ✅ 建議儲位（貼紙印的）= V6.9 S3 分配的位置
   ✅ 倉別一致
```

### 場景 B：V6.9 產生入庫單 → 瑞芳查詢
```
1. V6.9 移出端關單
   → 產生入庫單（32 行，每行有 來源單號）
   → 上傳 Google Sheet
   
2. 瑞芳卸貨端掃列印貼紙的 QR
   → 獲取 ALLY-20260614-A-1-001
   
3. 在 Sheet「入庫單」查詢：
   SELECT * WHERE 來源單號='ALLY-20260614-A-1-001'
   
4. 驗證：
   ✅ 找到 1 行以上（多 SKU 多行）
   ✅ 所有行的 虛擬倉 相同（或根據品項不同）
   ✅ 數量 = V6.9 掃描的數量
```

### 場景 C：儲位預配一致性
```
1. Tool① 讀 Google Sheet「儲位清單」
   → 為 BA 走道配 32 個儲位
   → 貼紙印上「建議儲位」
   
2. V6.9 移出端同樣走道
   → S3 封板時分配儲位
   
3. 驗證：
   ✅ 儲位順序相同（第1板→BA001-01, 第2板→BA001-02）
   ✅ V6.9 可覆寫（若現場被佔）→ 記錄「最終儲位」
```
